# 更新日志

## v2.14 — 学历证书核验

### 新增功能

- **新增学历证书核验**：支持批量导入毕业证书图片或 PDF，识别姓名、证书编号、学校和专业，并通过 Chrome 打开学信网完成批量查询、验证码识别与失败重试
- **新增独立核验工具**：学历核验功能可单独运行和打包，使用固定 AI 服务及独立 Chrome 数据目录，无需启动 BOSS 简历筛选器

## v2.13.1 — 打招呼状态跟踪与 AI 评估稳定性提升

### 体验优化

- **打招呼状态跟踪更可靠**：发送结果无法确认时记录为待确认状态，避免重复发送；后续扫描自动跳过待确认的候选人，核实后可继续操作
- **连续发送异常时自动停止**：打招呼连续多次结果不明确时自动暂停，提醒人工核实后再继续，避免批量无效操作
- **打招呼确认等待更充分**：延长发送后的状态确认等待时间，减少因页面响应慢导致的误判
- **AI 评估更稳定**：中转服务自动降低并发数，减少网关拥塞导致的失败；网络异常采用阶梯式重试间隔，恢复成功率更高

## v2.13 — AI 模型兼容与批量候选人操作

### 新增功能

- **多选批量处理候选人**：筛选结果页和明细窗口支持多选候选人，右键可批量打招呼、移除或导出；批量沟通会按候选人发送条件分组处理并保留执行结果
- **扩展 AI 服务兼容范围**：支持 Anthropic、Azure OpenAI 和常见 OpenAI 兼容服务，模型连接测试会验证能否生成程序可识别的评估结果

### 体验优化

- **候选人信息补全范围扩大**：单次最多补全 20 页候选人信息，提高大量候选人场景下的资料完整度
- **扫描完整性提示更明确**：达到扫描或信息补全上限时，如最后一轮仍有新增或有效命中，日志会提示结果可能不完整
- **AI 评估统计更透明**：评估结束显示成功数、失败数和成功率，失败率过高时提醒人工复核
- **模型兼容性验证更可靠**：连接测试会实际验证评估结果格式，并显示完整兼容或兼容模式；不支持结构化输出的模型会自动降级处理
- **筛选完成口径更准确**：分别显示规则筛选通过人数、AI 复核淘汰人数和最终保留人数，避免将最终人数误写成规则通过人数
- **已保存模型列表优化**：模型名称、服务商、状态、Base URL 四列始终显示，Base URL 过长时鼠标悬停显示完整内容；窗口最大化时列宽自动拉伸
- **批量操作反馈更完整**：模型测试和批量沟通会显示响应时间、停止状态，以及认证失败、配额超限、免费额度不足或模型未开通等原因，方便定位配置或页面问题
- **统招本科识别更准**：本科就读起止时间满 4 年的候选人自动通过学历硬条件，不再因学历形式疑似非统招误进人工确认
- **在职候选人减少人工确认**：求职状态为在职的候选人直接通过状态检查，不再标记为需人工确认

### 问题修复

- **修复学历经历误判**：第一学历明确为大专或专科后取得本科、专升本及本科教育不超过三年的候选人按岗位硬条件淘汰，不再误进入人工确认
- **修复 AI 返回格式异常导致批量评估失败**：通义千问评估改用稳定的参数化结果返回，并关闭不必要的深度思考；其他模型仍保留格式修复和纠错重试
- **修复部分服务商协议不兼容**：支持 Anthropic 和 Azure 的接口差异，统一处理 OpenAI 兼容服务，并按具体模型保存已验证的输出能力
- **修复启动时偶发页面断线误报**：浏览器刷新或重建页面导致旧连接失效时，跳过本次选择器检查并等待自动重连，不再误报选择器异常
- **修复剩余沟通次数被误判为已用完**：收紧沟通限制提示关键词，只在页面出现明确的次数耗尽提示，或可见升级弹窗同时包含沟通次数说明时停止发送；“今日剩余 2 次”等正常额度提示不再触发上限警告
- **修复点击打招呼后直接记为成功**：列表页发送后会确认按钮变为“继续沟通”或出现明确成功提示；页面状态无法确认时停止本轮发送且不写入已沟通状态，避免漏记或重复发送
- **修复已保存模型状态不显示**：首次配置或切换模型后，已验证的兼容状态可能未及时刷新到模型列表
- **修复切换结果页时偶发崩溃**：结果页面尚未就绪时触发刷新会导致程序异常退出

## v2.12.2 — 打招呼状态可靠性与结果页优化

### 新增功能

- **候选人明细支持完整操作菜单**：首页和统计明细窗口可按姓名、匹配分查看详情，支持导入简历做二次评估、更新跟进、标记反馈、手动打招呼、加入黑名单、移除或导出候选人

### 体验优化

- **筛选结果统计更直观**：统计卡片改为强烈推荐、推荐、待定、已打招呼四类，首页和数据统计页统一使用“通过筛选”口径和图标
- **AI 评估日志更紧凑**：每位候选人的分数变化、结论和理由摘要集中在一行展示，完整评估理由仍保留在候选人详情和 Excel 中
- **上下文保存状态更明确**：首次获取候选人发送信息时显示“已保存”，已有信息成功替换时显示“已刷新”
- **重复扫描提示更准确**：筛除当前岗位已沟通候选人后，日志改为显示“本轮待评估”人数，不再将历史未沟通候选人误称为新增
- **打招呼准备更充分**：优先为待定、需人工确认和超出单轮自动沟通上限的候选人保存发送所需信息，后续手动沟通更顺畅
- **状态提示更易理解**：结果表只展示业务状态；需要切换推荐页面时会说明原因，点击“确定”后继续，不再暴露内部技术信息

### 问题修复

- **修复硬条件信息不足时误判通过**：工作经验无法确认、只有本科层级但缺少统招证据时改为“需人工确认”，明确应届、经验不足或非统招学历时直接淘汰
- **修复 AI 判定淘汰后仍保留候选人**：AI 发现的硬条件问题必须提供原文证据并通过规则复核；确认淘汰或评分低于门槛后不再进入后续沟通队列
- **修复打招呼成功后状态偶尔丢失**：发送成功后立即保存本地状态，并在扫描结束或中断时合并最新数据，避免旧数据覆盖新状态
- **修复浏览器连接并发冲突**：自动检测和手动沟通同时连接浏览器时改为串行处理，避免偶发连接异常
- **修复未打开运行页时日志处理异常**：运行日志控件尚未创建时保留待显示内容，不再报错或丢失日志

## v2.11.4 — 三层扫描逻辑重构

### 体验优化

- **扫描节奏全面放宽**：API 补全预算从 80 提高到 160（默认最多补全 8 页）；单轮打招呼上限从 20 人放宽到 50 人，批次大小从 6 人放宽到 10 人；批次暂停和 API 分页间隔缩短，整体节奏更接近熟练人工操作
- **移除安全扫描开关**：三层扫描逻辑统一后不再区分安全扫描和 API 直调两种模式，运行控制页移除"安全扫描"复选框和"读取上限"输入框
- **岗位校验更严格**：刷新后岗位标识变化时弹窗提醒用户切回目标岗位，不再静默继续扫描；job_id 提取支持更多参数名，title fallback 采用保守判断

### 问题修复

- **修复无法打招呼和筛选的问题**：上一版本 API 优先的提取链路会导致候选人数据不完整，自动和手动打招呼均无法正常执行；重构为三层扫描逻辑（listener 补全首屏 → DOM 滚动建集合 → API 后置补全结构化字段），确保自动打招呼和筛选只作用于页面实际可见的候选人

## v2.11.3 — DOM 可点击扫描与页面跑偏检测

### 新增功能

- **结构化补全说明**：图形界面运行控制页固定展示 `listener + refresh → DOM → API 兜底` 的结构化补全策略，不再暴露"安全扫描"开关；运行日志只记录 listener 捕获数量和 DOM 命中数量
- **API 补全预算**：API 直调仅用于补全 DOM 已出现候选人的结构化字段，默认补全预算从 80 提高到 160，对应默认最多补全 8 页；补全页数按 `max_candidates` 预算推导但上限为 10 页，按 `page_num` range 范围逐页翻取且页间保留随机间隔
- **页面跑偏检测**：扫描和打招呼前自动检查浏览器是否仍在推荐牛人页面，页面被意外切换时停止本轮运行并提醒切回

### 体验优化

- **提取链路默认更稳**：默认先用 listener + refresh 捕获同页 API 结构化数据，再用 DOM 滚动确认可点击候选人，最后只对仍缺结构化字段的 DOM 候选人做 API 兜底；结构化数据只按 `geek_id` 合并，API 返回的额外候选人不会进入筛选和自动打招呼队列
- **DOM 扫描下节奏放宽**：DOM 可点击扫描成为主路径后，单轮自动打招呼上限从 20 人放宽到 50 人，批次大小从 6 人放宽到 10 人；批次暂停缩短为约 3-8 秒随机间隔，API 兜底分页间隔缩短为约 2-4 秒
- **页面上下文更一致**：listener + refresh 后会校验岗位标识，发现刷新后岗位变化会提示切回目标岗位；验证码和页面提示通过 `captcha_callback` / `notice_callback` 传递，等待循环使用 `done.wait(timeout=0.5)` 保持界面可停止；API 遇到 403/412/429 等风控状态只停止结构化补全，不丢弃已通过 DOM 提取的候选人
- **结构化日志更聚焦**：运行日志只保留 listener/API 捕获数量、命中 DOM 数量和分页兜底结果，清除接口地址、jobId、identity、API URL 探测等临时诊断输出
- **运行控制页简化**：移除旧安全扫描开关、读取上限输入框、读取上限滚轮处理（mousewheel）和安全扫描联动控件（removed def `_update_safe_scan_controls`），界面只保留 DOM 滚动轮次、岗位选择、自动打招呼和 AI 评估
- **更新日志远端修正**：更新日志弹窗现在从远端 Release Notes 获取当前版本说明，本地内容先显示、后台自动更新；读取失败时静默保留本地内容

## v2.11.2 — 反风控增强

### 新增功能

- **安全验证弹窗处理**：打招呼时自动检测验证码/安全验证弹窗，提示用户手动完成验证；验证完成后停止本轮自动打招呼，避免连续动作继续触发风控
- **自动打招呼上限提醒**：每轮自动打招呼最多 20 人，达到上限时弹窗提示已处理人数、剩余人数，以及下次扫描会自动跳过已打招呼候选人继续处理
- **验证码诊断包**：遇到验证码时自动保存诊断 JSON，并尽量保存页面截图，方便后续根据用户现场分析弹窗形态和选择器变化

### 体验优化

- **候选人提取链路更保守**：优先使用 API 直调分页；只有 API 直调失败后才启动 API listener 并刷新页面；仍失败时再回退 DOM 提取，减少不必要的刷新和监听动作
- **API 风控状态立即停止**：接口返回 403、412、429 等明确风控状态时立即停止本轮扫描，不再继续刷新页面或 DOM 兜底，避免扩大风险
- **API 分页和打招呼节奏放缓**：API 分页增加随机间隔；自动打招呼按批次暂停，整体节奏更接近人工操作
- **打招呼顺序更稳定**：自动打招呼按页面扫描顺序点击，不再按评分重排点击，降低虚拟列表大幅跳转带来的失败和风控风险
- **验证码识别范围扩展**：补充极验、captcha、verify 等常见安全验证选择器，提高弹窗识别覆盖率
- **更新日志远端修正**：更新日志弹窗先显示本地内置内容，再后台读取当前版本远端 Release Notes；读取成功后自动替换当前版本说明，失败时静默保留本地内容

### 问题修复

- **AI 评估状态显示修复**：运行控制页首次打开时 AI 评估状态显示不准确，现在正确反映 API Key 配置状态

## v2.11.1 — 界面启动与稳定性补丁

### 新增功能

- **简历导入格式扩展**：简历二次评估新增 TXT、Markdown、RTF、HTML 格式支持，纯文本和 HTML 无需额外依赖，RTF 需安装 striprtf（仅 22KB）
- **统计时间筛选扩展**：数据统计看板新增"本月"时间范围，方便按月查看筛选效果

### 体验优化

- **启动和切页更顺畅**：启动后只创建首页，运行控制、筛选结果、统计和模型配置页按需创建并错峰预热，减少首次进入重页面的卡顿
- **筛选结果页体验优化**：筛选结果页后台预热，真实点击时优先复用已创建页面；日期筛选控件保留下拉日历，不再退化成普通文本输入
- **模型配置页减少等待**：启动和首次打开模型配置页时不再同步等待系统钥匙串，API Key 改为后台读取后回填
- **窗口布局可配置**：新增 `ui_config.json`，用于统一管理窗口尺寸、页面间距、表格行高和部分控件宽度，方便不同屏幕下微调界面
- **全屏布局优化**：表单类页面（岗位配置、运行控制）内容宽度设上限，大屏居中显示不拉伸；结果页和统计页保持全宽利用
- **高窗口自动扩展行数**：窗口高度接近全屏时，岗位配置页的招聘需求文本框、技能列表和已保存模型列表自动增加可见行数，减少滚动
- **页面切换更跟手**：切换页面时延迟加载数据（首页统计、筛选结果、统计看板），不再在切页瞬间阻塞主线程

### 问题修复

- **退出保存更稳**：异常退出、中断或停止时自动保存进度并导出 Excel；停止打招呼时立即响应，不再继续处理剩余候选人
- **筛选条件修复**：修正”面议”出现在非薪资字段时薪资判断被跳过的问题；岗位未设年龄上限时不再默认限制 35 岁；硕士/博士学历含非统招关键词时标记待人工确认，不再无条件通过
