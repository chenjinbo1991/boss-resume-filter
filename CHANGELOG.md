# 更新日志

## v2.8.12 — 界面细节打磨 + 筛选默认值优化

### 体验优化
- **筛选结果日期默认一周**：日期过滤默认显示最近一周的结果（起始一周前、终止今天），重置按钮也恢复为这个范围，减少手动选日期的操作

### 问题修复
- **弹窗背景色统一**：修复"选择模型"和"清空候选人"弹窗在 macOS 上文字和按钮出现白色方块的问题；Windows 不受影响
- **升级后启动兼容**：修复部分电脑升级后因日历控件缺失导致 App 启动报错的问题；打包前也会提前检查日历依赖是否完整，并校验依赖清单没有遗漏

## v2.8.11 — 筛选结果增强 + 需求解析优化

### 新增功能
- **筛选结果日期过滤**：筛选结果页新增日期范围过滤，点击日历控件选择开始和结束日期，结合岗位下拉框实现双重过滤，快速定位特定时间段的筛选结果；两个日期控件互斥关闭，避免同时弹出；终止日期自动校验不小于起始日期、两个日期均不能超过今天

### 体验优化
- **卡片视觉统一**：所有功能卡片标题行增加蓝色竖线装饰，运行日志区域统一为卡片风格，与浏览器状态卡片视觉一致
- **明细弹窗优化**：标题和统计信息字号更紧凑，表格列头加粗，弹窗背景与页面统一，消除白色方块；右键菜单字体缩小更精致
- **候选人详情窗口放大**：查看详情窗口扩大，内容显示更充分
- **需求输入框占位提示**：岗位配置页输入框空时显示灰色提示文字，点击自动消失、清空自动恢复
- **页面切换更流畅**：优化页面切换性能，切换时不再有明显卡顿
- **字体统一**：岗位配置编辑区技能名称字体与页面其他区域保持一致
- **招聘需求解析更智能**：支持更多薪资格式（年薪、无标签、单边下限）、年龄格式（不超过、周岁以内、≤符号）、经验写法（中文数字、至少/不低于）；非标准段落标题（岗位职责、任职要求等）也能正确识别；从网页或微信复制的内容（全角字符、emoji）不再解析失败

### 问题修复
- **Windows 自动更新更可靠**：更新失败时会自动重试一次，降低更新中断的风险；同时清理残留临时文件，避免旧版本干扰
- **技能关键词误匹配修复**：修复 Go、API 等短词误匹配 Google、RAPID 等无关单词的问题
- **技能权重分配修复**：修复"精通"、"优先"、"熟练"等上下文关键词无法识别导致技能权重始终为默认值的问题
