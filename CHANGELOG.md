# 更新日志

## v2.7 — LLM 智能评估 + 数据统计看板 + 选择器健壮性加固

### 新增功能
- **LLM 智能评估**：对通过筛选的候选人调用 LLM 做二次评估，返回 ±10 分调整值，直接影响推荐等级和打招呼决策；新增 `llm_eval.py` 模块
- **数据统计看板**：侧边栏新增「数据统计」页，按岗位维度聚合筛选和打招呼数据，支持岗位过滤和时间范围过滤（今天/本周/全部）
- **选择器自动检查**：浏览器连接到推荐牛人页面后自动执行选择器健康检查，有异常弹窗提醒；移除手动「选择器检查」按钮

### 行为优化
- **淘汰候选人不持久化**：`save_candidates_all()` 在保存前过滤低于 55 分的淘汰候选人，不再写入 `candidates_all.json`；`export_to_excel()` 已有相同过滤逻辑，两处口径统一
- **智能滚动定位**：`send_greeting_on_list_page` 从单次 300px 滚动重试改为三阶段智能搜索（当前位置 → 滚到顶部 → 逐步向下 800px/步，最多 40 轮），同时滚动 window 和列表容器元素，确保补打招呼和右键打招呼在候选人不在可视区时也能稳定定位
- **滚动扫描性能优化**：新增 `_extract_cards_batch()` 函数，单次 JS 调用批量提取所有候选人卡片的 geek_id、innerText、姓名（替代逐卡片的 N+1 网络调用，从每轮 ~90 次降到 ~4 次）；验证码检测从每轮改为每 3 轮一次；到底检测改用滚动位置判断（`get_frame_scroll_info`）替代文本查找；连续空轮次阈值从 10 降到 5；滚动间隔从 0.5s 放宽到 0.8s 更像人类操作。整体每轮耗时从 1.5-2.5s 降到 0.5-0.8s，提速约 3 倍
- **验证码等待恢复机制**：检测到安全验证弹窗后不再直接停止扫描，改为暂停等待用户手动完成验证（每 3 秒轮询、最长 5 分钟），验证完成后自动恢复滚动和打招呼；支持 stop_event 中断
- **评分权重调整**：技能匹配权重从 35 分提升到 50 分，基础分从 30 降到 25，经验超额从 20（4/年）降到 15（3/年），学历从 15 降到 10；0 技能匹配的候选人不再能达到 55 分（最高 50 分）
- **页面选择器外部化**：所有与 BOSS 直聘 DOM 交互的 CSS/XPath 选择器和关键词列表集中到 `selectors.json`，BOSS 前端结构变化时只改配置文件，无需改代码
- **统计数据优质率**：岗位明细表的"通过率"列改为"优质率"（强烈推荐+推荐占比），打招呼率同步只显示百分比

### UI 改进
- **更新日志页面重构**：从单文本框改为左右分栏——左侧版本历史列表（Listbox），右侧版本详情（Text）；点击版本标签切换内容；标题/分类/条目分级样式渲染

- **验证码弹窗通知**：检测到安全验证弹窗时，GUI 弹出对话框通知用户（显示匹配关键词），提供「继续等待」和「跳过验证」两个选项，避免误报时无任何反馈；回调通过 `threading.Event` 安全跨线程
- **运行页 AI 评估开关**：新增"启用 AI 辅助评估"复选框，默认关闭；开启后日志显示评估进度
- **筛选结果表新增列**：新增「AI评估」列，显示 LLM 调整值（如 +7 / -3 / —）

### UI 改进
- **必要条件 UI 重构**：JSON 输入 → 下拉框选择类型（简单匹配/OR满足任一/AND全部满足）+ 逗号分隔关键词，无需手写 JSON

- **Chrome 启动异常处理**：`FileNotFoundError`（Chrome 未安装）和 `chrome`/`errno` 关键词异常（Chrome 安装异常）分类处理，替代宽泛的超时提示

- **明文 API Key 泄露**：`load_api_config()` 将 keyring 中的明文 key 注入了 `saved_models` 字典（内存原地修改），导致后续任何一次保存/切换/删除模型操作都将明文 key 写回 `api_config.json` 磁盘文件；修复：不再注入 key，改为按需从 keyring 读取；新增 `_sanitize_config_for_save()` 防御层剥离顶层和嵌套 `api_key`；三个写入路径 + `migrate_keys.py` 全部使用该辅助方法
