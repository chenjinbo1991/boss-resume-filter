# BOSS 简历筛选器 - 图形界面版使用说明

## 启动方式

### 方式 1：双击启动
双击 `gui.bat` 文件即可启动图形界面。

### 方式 2：命令行启动
```bash
python gui_main.py
```

## 界面功能

左侧导航栏共 6 个页面（首页、岗位配置、运行控制、筛选结果、学历核验、数据统计），底部另有系统设置入口。

### 1. 首页

启动后默认显示的页面，展示全局统计数据。

**岗位过滤**：选择单个岗位或"全部岗位"，统计数字自动刷新

**统计卡片**（点击数字弹出明细弹窗）：
- **通过筛选**：≥55 分的候选人总数
- **强烈推荐**：≥75 分的候选人数
- **推荐**：65-74 分的候选人数
- **已打招呼**：已发送消息的候选人数

### 2. 岗位配置

**选择岗位**：从下拉列表选择已配置的岗位

**新建岗位**：点击后清空表单，输入新岗位信息

**删除岗位**：删除当前选中的岗位配置

**岗位配置表单**：
- **岗位名称**：岗位的名称（如：高级 Java 工程师）
- **最低学历**：要求的最低学历（不限/高中/中专/大专/本科/硕士/博士）
- **最低经验**：要求的最少工作年限
- **最大年龄**：候选人年龄上限（默认 35 岁），留空不限制
- **薪资范围**：岗位薪资预算范围（如 12K ~ 15K），留空不限制
- **工作地点**：自由输入，支持多地点（如：南京、上海/杭州）
- **技能关键词**：逗号分隔的技能列表（如：Spring, MySQL, Redis）
- **必要条件**：选择条件类型（简单匹配 / OR满足任一 / AND全部满足），输入逗号分隔的关键词（支持全角/半角逗号）

**招聘需求解析**：

- 在输入框中粘贴招聘需求，点击"解析招聘需求"自动提取：岗位名称、经验、学历、年龄上限、薪资范围、工作地点、技能关键词、优先项和必要条件
- 解析流程先用本地正则生成基础结果；如果已配置可用的大模型和 API Key，会自动启用 AI 增强解析，补充和修正基本信息、技能关键词、优先项、必要条件；未配置或调用失败时自动回落到本地正则解析
- "全栈开发经验优先"、"数据库运维经验优先"、"证券行业优先"等优先项会进入 `preferred_keywords`，作为排序加分项，不会稀释普通技能关键词匹配分
- "金融投资行业经验，债券、基金、期货、期权等"这类必要条件会尽量解析为 OR 条件，候选人满足任一方向即可通过
- 点击"招聘需求示例"按钮可一键填充模板文本，方便按格式修改（仅在"新建岗位"模式下可用）
- 新建岗位流程有呼吸动画提示引导：填入需求→解析需求→检查结果→保存配置

**导入/导出配置**：可以从文件加载配置或保存配置到文件

### 3. 大模型配置

配置 AI 辅助评估使用的大模型。

**服务商选择**：支持 qwen、deepseek、kimi、zhipu、minimax、xiaomi、stepfun、openai、anthropic、custom 等 10 种服务商

**API Key**：
- 输入 API Key 后可点击眼睛图标切换明文/密文显示
- Key 加密存储在系统钥匙串中，配置文件不含明文
- 同一服务商的所有模型共享一个 Key

**模型选择**：
- 输入 API Key 后双击可动态获取可用模型列表
- 也可手动输入模型名称

**测试连接**：验证 API Key 和模型是否可用

### 4. 运行

**浏览器状态**：显示 Chrome 浏览器连接状态（未连接/已连接），点击"检测/连接浏览器"按钮进行连接。连接到 BOSS 直聘推荐牛人页面后，程序会自动检查 `selectors.json` 中的选择器是否正常工作，如有异常会弹窗提醒

**滚动轮次**：扫描时自动滚动的次数，默认 100，推荐 50-200。支持鼠标滚轮微调

**候选人提取**：程序先补全页面已有的结构化信息，再通过页面滚动确认可见候选人；必要时补全已在页面出现的候选人信息。运行控制页不再提供“安全扫描”和“读取上限”开关

**AI 辅助评估**：勾选后对通过筛选的候选人调用 LLM 做二次评分（±10 分调整），需先在「大模型配置」中配置好 API Key

**选择岗位**：从下拉列表选择要处理的岗位（"全部岗位"依次处理所有岗位）

**自动打招呼**：
- **不打招呼（仅筛选）**：只筛选候选人，不发送消息
- **仅强烈推荐**：只给 75 分以上的候选人打招呼（默认）
- **强烈推荐 + 推荐**：给 65 分以上的候选人打招呼

**筛选进度**：实时显示当前扫描/筛选进度百分比

**控制按钮**：
- **开始**：启动运行
- **停止**：立即停止运行（停止信号穿透所有循环，自动保存进度）

**运行日志**：实时显示程序运行过程中的日志信息，支持清空日志

### 5. 筛选结果

**统计信息**：总人数、通过人数、已打招呼人数

**候选人列表**：表格显示候选人信息（≥55 分，含强烈推荐、推荐、待定三个等级），已加入黑名单的候选人默认不显示
- 姓名、工作经验、期望薪资、匹配分、推荐指数、AI评估（开启 LLM 评估时显示调整值）、状态（含跟进状态、人工反馈标记、黑名单标记和风险确认标记）、技能匹配等
- 待定候选人（55-64 分）以浅色背景标记，支持右键打招呼

**操作按钮**：
- **刷新结果**：从 candidates_all.json 加载最新数据
- **导出 Excel**：生成 candidates_all.xlsx 文件
- **打开 JSON**：用默认程序打开 candidates_all.json 文件
- **清空候选人**：清空候选人数据（当前岗位或全部岗位），操作前自动备份到 candidates_all.json.bak；默认勾选"保留已打招呼的候选人"，清空时保留已沟通过的记录；已屏蔽候选人始终保留为黑名单

### 6. 候选人明细弹窗

点击筛选结果页的统计指标数字（强烈推荐、推荐、待定、已打招呼），弹出候选人明细窗口。

**弹窗内容**：
- 标题显示分类名称（强烈推荐 / 推荐 / 待定 / 已打招呼）
- 统计行显示总人数和已打招呼人数
- 表格列：姓名、工作年限、薪资、匹配分、推荐指数（强烈推荐/推荐/待定）、状态（已招呼/未招呼）、技能匹配

**模态窗口**：明细弹窗打开期间，主窗口（包括左侧导航栏）不可操作，必须先关闭弹窗后才能继续操作主界面

**右键菜单**（在表格行上右键）：

**单选模式**（右键单个候选人）：
- **查看详情**：弹出候选人详情窗口，结构化展示核心信息（年龄/工作年限/薪资/求职状态）、教育信息（学校·专业·学历）、评分拆解（基础/技能/经验/学历/优先项各项分值）、评分解释、命中证据、AI 评估（评估理由、调整值、原始分数、模型名称）、风险提示（如有）、跟进状态（状态/备注/时间）、人工反馈（状态/备注/时间）、黑名单状态（如有）、技能匹配详情（含匹配数/总数）、候选人摘要
- **导入简历**：选择候选人的 PDF/Word 简历后，基于完整简历做二次评估；结果会写入候选人数据、详情页和 Excel 导出
- **撤销简历评估**：已导入简历并完成二次评估的候选人可撤销评估，清空简历文件和二次评估结果，分数回退到一次评估状态
- **打招呼**：对选中的未打招呼候选人单独发送消息（仅对状态为"未招呼"的候选人显示）。程序会自动滚动搜索定位候选人卡片（无需手动滚动浏览器页面）；发送后只有页面明确变为“继续沟通”或出现成功提示时才更新状态，无法确认时会提醒人工核实
- **更新跟进**：更新候选人的跟进状态（未沟通/已打招呼/已回复/待约面/已约面/不合适/已归档）和备注，跟进状态会写入候选人数据、详情页和 Excel 摘要
- **标记反馈**：标记候选人的人工反馈状态（合适/误推/误杀/放弃）和备注，反馈会写入候选人数据、详情页和 Excel 摘要
- **加入黑名单**：记录屏蔽原因并按 `geek_id` 跨岗位屏蔽该候选人；后续扫描、统计和 Excel 导出都会跳过
- **移除此人**：从列表和 `candidates_all.json` 中删除该候选人，弹窗保持打开并即时更新统计数字
- **导出选中**：将选中的候选人导出为 Excel 文件（支持多选）

**多选模式**（Ctrl/Shift 多选后右键）：
- **批量打招呼**：批量向选中的未打招呼候选人发送消息。有打招呼上下文的候选人可直接发送，无上下文的候选人需要浏览器在对应岗位的推荐牛人页面；列表页发送结果无法确认时停止本轮并提示核实，分组处理时保持 2-4 秒间隔
- **移除选中**：批量删除选中的候选人
- **导出选中**：批量导出选中的候选人到 Excel

**首页统计卡片**也支持同样的明细弹窗和右键菜单（点击"通过筛选"、"强烈推荐"等卡片数字）。

### 7. 学历核验

- 支持一次导入多张 JPG、JPEG、PNG、BMP、WEBP 图片或 PDF 文件的毕业证书
- 待核验队列独立显示每张证书的姓名、证书编号、学校、专业和处理状态，点击队列可切换当前证书
- 队列支持 Ctrl/Shift 多选；点击”识别证书”可同时识别选中证书，最多 3 路并发
- 多选后点”移除当前”会移除所有选中项；单选时只移除当前项
- 图片预览先按 EXIF 中明确的方向信息纠正；没有可靠 EXIF 信息时保持原样
- 图片模型通过四方向对照图判断是否还需旋转 90°、180°或270°；方向置信度不足时保持当前方向
- 预览卡片标题栏右侧保留”顺转 90°”作为人工兜底；用户点击后锁定人工方向，重新识别不会覆盖
- PDF 文件通过提取文本层识别（走文本模型，不渲染图片），预览区只显示文字提示；扫描件 PDF 无文本层时会提示”请导出为图片后导入”，改用图片格式即可正常识别
- 右侧识别结果仍只显示姓名和证书编号，学校、专业只在队列中展示
- 右键队列行可识别选中证书或将选中项移出队列
- 学信网验证仍逐张进行，避免候选人信息错配
- 使用当前配置的大模型识别姓名和证书编号（图片走视觉模型，PDF 走文本模型）
- 识别结果必须由 HR 人工确认或修正后，再打开学信网填写
- 系统填写姓名和证书编号并勾选用户协议；图片验证码和手机扫码由 HR 人工完成
- 识别时图片或 PDF 内容会发送给当前配置的 AI 服务，请在取得候选人授权后使用

### 8. 数据统计

侧边栏第 6 个导航项，按岗位维度聚合历史筛选和打招呼数据。

**过滤条件**：
- **岗位过滤**：选择单个岗位或"全部岗位"
- **时间范围**：今天 / 本周 / 全部

**汇总卡片**：
- 通过筛选人数、强烈推荐人数、推荐人数、已打招呼人数

**岗位明细表格**（9 列精简布局）：
- 岗位名称、筛选分布（总数+强推/推荐/待定）、已打招呼(率)、已反馈、合适率、误推率、已回复(率)、已约面(率)、平均分
- 低于 55 分的候选人不计入统计
- 合适率/误推率只按有效人工反馈计算（合适/误推/误杀/放弃）
- 已回复/已约面列内嵌百分比，分别按已打招呼及后续状态、已回复及后续状态计算

点击"刷新"按钮或切换过滤条件时自动重新统计。


## 使用流程

### 初次使用

1. **配置岗位**：
   - 点击"新建岗位"
   - 输入岗位名称、经验、学历要求
   - 输入技能关键词（逗号分隔）
   - 点击"保存配置"

2. **开始扫描**：
   - 选择要处理的岗位（或"全部岗位"）
   - 设置滚动轮次和打招呼等级
   - 点击"开始"
   - 手动导航到 BOSS 直聘推荐页面
   - 等待程序自动运行

3. **查看结果**：
   - 运行完成后，点击"刷新结果"
   - 查看候选人列表
   - 点击"导出 Excel"生成报表

## 注意事项

1. **浏览器要求**：需要安装 Chrome 浏览器（程序可自动启动 Chrome 并导航到推荐页面）
2. **手动导航**：如果浏览器未自动导航到推荐页面，请手动导航到 BOSS 直聘推荐页面
3. **数据备份**：定期备份 candidates_all.json 文件
4. **网络稳定**：保持网络连接稳定
5. **安全验证弹窗**：运行过程中如果 BOSS 直聘弹出滑块/图形验证码，程序会自动暂停并弹出对话框通知。在浏览器中手动完成验证后，点击对话框中的「是」继续等待验证完成；如果是误报（浏览器中没有看到验证码），点击「否」跳过等待继续运行

## 常见问题

**Q: 点击开始没有反应？**
A: 检查是否已安装 Chrome 浏览器，确保没有其他程序占用。

**Q: 日志显示乱码？**
A: 这是正常的，不影响功能使用。

**Q: 如何查看详细的候选人评分？**
A: 打开 candidates_all.json 文件，查看每个候选人的 match_score 字段。评分采用四维模型：基础25 + 技能(0~50) + 经验超额(0~15) + 学历档次(0~10)，总分范围 25-100。同时可查看 skill_matches 字段了解具体匹配的技能。
