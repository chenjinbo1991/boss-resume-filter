# BOSS 简历筛选器 - 图形界面版使用说明

## 启动方式

### 方式 1：双击启动
双击 `gui.bat` 文件即可启动图形界面。

### 方式 2：命令行启动
```bash
python gui_main.py
```

## 界面功能

### 1. 岗位配置区域

**选择岗位**：从下拉列表选择已配置的岗位

**新建岗位**：点击后清空表单，输入新岗位信息

**删除岗位**：删除当前选中的岗位配置

**岗位配置表单**：
- **岗位名称**：岗位的名称（如：高级 Java 工程师）
- **最低学历**：要求的最低学历（不限/高中/中专/大专/本科/硕士/博士）
- **最低经验**：要求的最少工作年限
- **最大年龄**：候选人年龄上限（默认 35 岁），留空不限制
- **薪资范围**：岗位薪资预算范围（如 12K ~ 15K），留空不限制
- **工作地点**：自由输入，支持多地点（如：南京、上海/杭州）
- **技能关键词**：逗号分隔的技能列表（如：Spring, MySQL, Redis）
- **必要条件**：选择条件类型（简单匹配 / OR满足任一 / AND全部满足），输入逗号分隔的关键词（支持全角/半角逗号）

**需求文档解析**：
- 在输入框中粘贴招聘需求文档，点击"解析并填充"自动提取：岗位名称、经验、学历、年龄上限、薪资范围、工作地点、技能关键词
- 点击"招聘需求示例"按钮可一键填充模板文本，方便按格式修改（仅在"新建岗位"模式下可用）

**导入/导出配置**：可以从文件加载配置或保存配置到文件

### 2. 运行控制区域

**浏览器状态**：显示 Chrome 浏览器连接状态（未连接/已连接），点击"检测/连接浏览器"按钮进行连接

**滚动轮次**：扫描时自动滚动的次数（默认 100，推荐 50-200），支持鼠标滚轮微调

**选择岗位**：从下拉列表选择要处理的岗位（"全部岗位"依次处理所有岗位）

**自动打招呼**：
- **不打招呼（仅筛选）**：只筛选候选人，不发送消息
- **仅强烈推荐**：只给 75 分以上的候选人打招呼（默认）
- **强烈推荐 + 推荐**：给 65 分以上的候选人打招呼

**筛选进度**：实时显示当前扫描/筛选进度百分比

**控制按钮**：
- **开始**：启动运行
- **停止**：立即停止运行（停止信号穿透所有循环，自动保存进度）

### 3. 运行日志区域

实时显示程序运行过程中的日志信息。

**清空日志**：清除所有日志内容

### 4. 筛选结果区域

**统计信息**：总人数、通过人数、已打招呼人数

**候选人列表**：表格显示候选人信息
- 姓名、匹配分、推荐指数、状态、技能匹配等

**操作按钮**：
- **刷新结果**：从 candidates_all.json 加载最新数据
- **导出 Excel**：生成 candidates_all.xlsx 文件
- **打开 JSON**：用默认程序打开 candidates_all.json 文件

## 使用流程

### 初次使用

1. **配置岗位**：
   - 点击"新建岗位"
   - 输入岗位名称、经验、学历要求
   - 输入技能关键词（逗号分隔）
   - 点击"保存配置"

2. **开始扫描**：
   - 选择要处理的岗位（或"全部岗位"）
   - 设置滚动轮次（默认 100）和打招呼等级
   - 点击"开始"
   - 手动导航到 BOSS 直聘推荐页面
   - 等待程序自动运行

3. **查看结果**：
   - 运行完成后，点击"刷新结果"
   - 查看候选人列表
   - 点击"导出 Excel"生成报表

## 注意事项

1. **浏览器要求**：需要安装 Chrome 浏览器（程序可自动启动 Chrome 并导航到推荐页面）
2. **手动导航**：如果浏览器未自动导航到推荐页面，请手动导航到 BOSS 直聘推荐页面
3. **数据备份**：定期备份 candidates_all.json 文件
4. **网络稳定**：保持网络连接稳定

## 常见问题

**Q: 点击开始没有反应？**
A: 检查是否已安装 Chrome 浏览器，确保没有其他程序占用。

**Q: 日志显示乱码？**
A: 这是正常的，不影响功能使用。

**Q: 如何查看详细的候选人评分？**
A: 打开 candidates_all.json 文件，查看每个候选人的 match_score 字段。评分采用四维模型：基础30 + 技能(0~35) + 经验超额(0~20) + 学历档次(0~15)，总分范围 30-100。同时可查看 skill_matches 字段了解具体匹配的技能。
